# SuperOPL Skill 交接文档（2026-05-12）

## 本次交接结论
SuperOPL skill 已完成真实 API 联调，不再是仅语法通过的原型。当前版本已可稳定执行读写追踪与基础知识检索。

## 核心变化（相对 2026-05-11）
1. 完成真实 API 适配（`{result, data}` 包装结构）
2. 新增标准化层：`Get-SuperOPLTasks`，统一 task 字段
3. 修复 StrictMode 下路径与 URI 拼接问题
4. 完成 setup/read/write/edit/track/report/sync/search 端到端验证
5. 新增 `scripts/opl_delete.ps1`（DELETE 需要 body.loginUser）

## 当前可用能力
- 读取 OPL、tasks、risks（risks 缺失时不报错）
- 创建 task/problem/measure
- 编辑 task（截止日期、状态、负责人、主题）
- 删除 task
- 自动追踪 overdue/upcoming
- 输出 Markdown 状态报告
- 同步本地知识库并按关键词检索历史条目

## 关键脚本说明
- `scripts/common.ps1`
  - `Get-SuperOPLData`: 提取真实 data 层
  - `Get-SuperOPLTasks`: task 结构归一化
- `scripts/opl_delete.ps1`
  - DELETE 请求体自动注入 `loginUser`
